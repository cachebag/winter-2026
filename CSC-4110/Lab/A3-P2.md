# Use-Case: Book Study Room

  

## 1. Brief Description

  

This use case describes how a WSU Student uses the warrior-bot CLI to book a study room at a Wayne State University library or building.

  

## 2. Actors

  

### 2.1. WSU Student

The primary user who wants to reserve a study room on campus.

  

### 2.2. WSU Booking System

The university system that manages room availability and reservations.

  

## 3. Preconditions

  

- The warrior-bot CLI tool is installed on the user's device.

- There is an active network connection to WSU resources.

- The user has valid WSU credentials (student ID, email).

  

## 4. Basic Flow of Events

  

1. The use case begins when the WSU Student enters the `wb book` command.

2. The CLI displays available location types (libraries, buildings).

3. The WSU Student selects a location type.

4. The CLI displays a list of available locations (e.g., Purdy/Kresge Library, Undergraduate Library, State Hall).

5. The WSU Student selects a specific location.

6. The CLI queries the WSU Booking System for available study rooms at that location.

7. The CLI displays available study rooms with time slots.

8. The WSU Student selects a study room and time slot.

9. The CLI prompts for authentication (student ID, email, and/or full name).

10. The WSU Student enters their authentication information.

11. The booking request is sent to the WSU Booking System.

12. The WSU Booking System confirms the reservation.

13. The CLI displays a confirmation message with booking details.

14. If an email was provided, a confirmation is sent to the user's email.

15. The use case ends successfully.

  

## 5. Alternative Flows

  

### 5.1. No Available Rooms

  

If in step 7 of the basic flow no study rooms are available at the selected location, then:

  

1. The CLI displays the message "No study rooms available at this location for the selected time."

2. The CLI prompts the user to select a different location or time.

3. The use case resumes at step 4.

  

### 5.2. Invalid Location Selection

  

If in step 5 of the basic flow the WSU Student enters an invalid location, then:

  

1. The CLI displays the message "Invalid location – please select from the list."

2. The use case resumes at step 4.

  

### 5.3. Authentication Failure

  

If in step 10 of the basic flow the authentication information is invalid or incomplete, then:

  

1. The CLI displays the message "Invalid credentials – please verify your student ID and email."

2. The use case resumes at step 9.

  

### 5.4. Booking System Unavailable

  

If in step 11 of the basic flow there is no response from the WSU Booking System within 15 seconds, then:

  

1. The CLI will retry up to three times.

2. If there is still no response, the CLI displays the message "Booking system unavailable – please try again later."

3. The use case ends with a failure condition.

  

### 5.5. Room No Longer Available

  

If in step 12 of the basic flow the WSU Booking System indicates the room is no longer available (booked by another user), then:

  

1. The CLI displays the message "This room was just booked by another user. Please select a different room."

2. The use case resumes at step 7.

  

### 5.6. Maximum Bookings Exceeded

  

If in step 12 of the basic flow the WSU Booking System indicates the user has reached their maximum number of active bookings, then:

  

1. The CLI displays the message "You have reached the maximum number of active bookings. Please cancel an existing booking first."

2. The use case ends with a failure condition.

  

### 5.7. Quit

  

If at any point prior to step 11 in the basic flow the WSU Student enters `quit` or `cancel`, then:

  

1. The CLI displays the message "Booking cancelled."

2. The use case ends.

  

### 5.8. List Available Spaces

  

If the WSU Student enters `wb book --list`, then:

  

1. The CLI displays all bookable locations organized by type (libraries, buildings).

2. The use case ends.

  

## 6. Post-conditions

  

### 6.1. Successful Completion

  

- The study room has been reserved for the WSU Student.

- A confirmation message is displayed with booking details (location, room, date, time).

- If an email was provided, a confirmation email is sent.

  

### 6.2. Failure Condition

  

- No booking is made.

- An appropriate error message is displayed to the user.

  

## 7. Special Requirements

  

- **[SR:BK-1]** The CLI shall display results within an average of 15 seconds.

- **[SR:BK-2]** The CLI shall support booking rooms at all WSU libraries (Purdy/Kresge, Undergraduate Library, Shiffman Medical Library, etc.).

- **[SR:BK-3]** The CLI shall support booking rooms at WSU buildings with study spaces (State Hall, etc.).

- **[SR:BK-4]** User authentication shall require at minimum: email OR student ID.

- **[SR:BK-5]** The CLI shall run on Windows, macOS, and Linux platforms.

  

## 8. Related Requirements

  

- **FR11**: Book command lists each library to choose to book

- **FR12**: Book command lists each building to choose to book study rooms

- **FR16**: Book command authentication for booking study rooms

- **NR2**: Results in at average 15 seconds
